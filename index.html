LLM Load Balancer

Ship it. We dare you.

shunt sits between your application and multiple LLM backends. It routes requests by model group, prefers backends with cached KV-state for repeated prefixes, and passes SSE streaming through unchanged. Swap your base URL — nothing else changes.

Download Source Code

Features

KV-cache reuse routing

Hashes the prompt prefix and routes to the backend that already has it cached. Cache hit means skipping up to 90% of prefill compute. No match? Falls back to least-connections.

OpenAI-compatible

Drop-in replacement. Supports /v1/chat/completions, /v1/completions, /v1/models, and /health. Your existing client libraries work without modification.

Health checking

Every backend gets polled at a configurable interval. Three consecutive failures marks it unhealthy and it stops receiving traffic until it recovers.

SSE passthrough

text/event-stream responses stream through without buffering. Chat completions work out of the box with streaming enabled.

Request queue + backpressure

If all backends are busy, requests queue up with a configurable cap and timeout. No silent drops.

Model-group routing

Each model gets its own backend pool. Route llama3 to one set of servers and mistral to another.

Quick Start

1. Download

curl -LO https://github.com/ShitwareIndustries/shunt/releases/latest/download/shunt-linux-x86_64
chmod +x shunt-linux-x86_64

2. Configure

# config.toml
[balancer]
listen_addr = "0.0.0.0:8080"

[[models]]
id = "primary"
address = "http://127.0.0.1:8081"
model = "llama3"

3. Run

./shunt-linux-x86_64 --config=config.toml

Point your OpenAI client at http://localhost:8080 and you are load balancing.
